## 一、服务器选型

- **GPU**：8× NVIDIA RTX 4090（24GB × 8 = 192GB 总显存）
- **CUDA Toolkit 版本**：**12.8**（必须和驱动匹配！）
- **NVIDIA 驱动**：≥ 570（驱动决定 CUDA 最高版本，`nvidia-smi` 右上角看）

> ⚠️ **最关键坑**：Docker 镜像预装的 CUDA Toolkit 版本可能和宿主机驱动不兼容。新机器到手第一件事：
> ```bash
> nvidia-smi  # 看右上角 "CUDA Version: X.X"
> nvcc --version  # 看 CUDA Toolkit 版本
> ```
> 两者必须匹配。如果 Toolkit 版本 > 驱动支持的版本，按下面 2.1 节降级。

## 二、系统环境配置

### 2.1 检查 CUDA 兼容性并降级（如需要）

```bash
# 查看驱动支持的最高 CUDA 版本
nvidia-smi | grep "CUDA Version"

# 查看当前 CUDA Toolkit 版本
nvcc --version

# 如果 nvcc 版本 > 驱动支持版本，需要降级 CUDA Toolkit
# 假设驱动是 570（支持 CUDA 12.8），但装了 CUDA 13.3：
```

如果 CUDA 版本不匹配，下载 CUDA 12.8 runfile 安装：

```bash
# 下载 CUDA 12.8（驱动 570 的匹配版本）
wget https://developer.download.nvidia.com/compute/cuda/12.8.1/local_installers/cuda_12.8.1_570.124.06_linux.run -O /tmp/cuda_12.8.run

# 安装（--silent 静默安装，--toolkit 只装 toolkit 不装驱动）
sh /tmp/cuda_12.8.run --silent --toolkit --toolkitpath=/usr/local/cuda-12.8

# 切换默认 CUDA
rm -f /usr/local/cuda && ln -s /usr/local/cuda-12.8 /usr/local/cuda

# 验证
/usr/local/cuda/bin/nvcc --version  # 应显示 12.8
```

### 2.2 修复 NVML 库版本不匹配（如需要）

如果 `nvidia-smi` 报 `Driver/library version mismatch`：

```bash
# 查看 NVML 库版本
ls -la /usr/lib/x86_64-linux-gnu/libnvidia-ml.so*

# 如果有多个版本（如 570 和 580），清除不匹配的包
sudo dpkg --purge --force-all <错误版本的包名>

# 确保 libnvidia-ml.so.1 指向驱动版本
sudo ln -sf /usr/lib/x86_64-linux-gnu/libnvidia-ml.so.570.124.06 /usr/lib/x86_64-linux-gnu/libnvidia-ml.so.1
```

### 3.1 InfiniLM 代码改动（共 1 个文件）

**`setup.py`**：`build_cpp_module()` 函数加 `XMAKE_ROOT=y` 环境变量，允许 root 运行 xmake。

```python
# 修改前（原版）
def build_cpp_module():
    subprocess.run(["xmake", "build", "_infinilm"], check=True)
    subprocess.run(["xmake", "install", "_infinilm"], check=True)

# 修改后
def build_cpp_module():
    import os
    env = {**os.environ, "XMAKE_ROOT": "y"}
    subprocess.run(["xmake", "build", "_infinilm"], check=True, env=env)
    subprocess.run(["xmake", "install", "_infinilm"], check=True, env=env)
```

### 3.2 InfiniCore 代码改动（共 2 个文件）

**（1）`scripts/install.py`**：两处修改

```python
# ① run_cmd() 函数加 XMAKE_ROOT=y
# 修改前
def run_cmd(cmd):
    subprocess.run(cmd, text=True, encoding="utf-8", check=True, shell=True)

# 修改后
def run_cmd(cmd):
    env = {**os.environ, "XMAKE_ROOT": "y"}
    subprocess.run(cmd, text=True, encoding="utf-8", check=True, shell=True, env=env)

# ② install() 函数给 xmake build 加 -j4 并行限制（避免 OOM）
# 修改前
def install(xmake_config_flags=""):
    ...
    run_cmd("xmake -y")
    run_cmd("xmake install -y")
    run_cmd("xmake build -y infiniop-test")
    ...

# 修改后
def install(xmake_config_flags=""):
    ...
    run_cmd("xmake -y -j4")           # build 加 -j4
    run_cmd("xmake install -y")       # install 不加 -j
    run_cmd("xmake build -y -j4 infiniop-test")
    ...
```

> 注意：`xmake install` 和 `xmake f` 命令**不支持** `-j` 参数，只有 `xmake build` 需要。

**（2）`setup.py`**：`run_xmake_build()` 函数加 `XMAKE_ROOT=y`

```python
# 修改前
def run_xmake_build():
    subprocess.run(["xmake", "build"], check=True)
    ...

# 修改后
def run_xmake_build():
    env = {**os.environ, "XMAKE_ROOT": "y"}
    subprocess.run(["xmake", "build"], check=True, env=env)
    ...
```

## 四、编译 InfiniCore

```bash
cd ~/difanrui/InfiniCore

# 一步完成：配置 + 编译 + 安装（必须加 GPU 参数！）
python3 scripts/install.py --nv-gpu=y --cuda=/usr/local/cuda

# 验证 GPU 后端是否编译进去了
find build -name "libinfiniop-nvidia.a"  # 存在 = GPU 后端已编译
find build -name "libinfinirt-nvidia.a"  # 存在 = GPU runtime 已编译
```

> ⚠️ **必须在 install.py 后加 `--nv-gpu=y --cuda=/usr/local/cuda`**，否则只编译 CPU 后端，GPU 推理会失败。

## 五、安装 InfiniLM

```bash
cd ~/difanrui/InfiniLM

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装 InfiniLM
pip install -e .

# 安装 InfiniCore Python 包
cd ~/difanrui/InfiniCore
pip install -e .

# 手动编译 Python 绑定并复制到正确位置
XMAKE_ROOT=y xmake build -y _infinicore
mkdir -p python/infinicore/lib/
touch python/infinicore/lib/__init__.py
cp build/linux/x86_64/release/_infinicore.cpython-*.so python/infinicore/lib/
```

> ⚠️ **不能跳过**最后一步。`pip install -e .` 不会自动编译 Python C++ 扩展的 `.so` 文件，需要手动 `xmake build _infinicore`。

## 十一、已知问题 & 解决方案速查

| 问题 | 原因 | 解决 |
|------|------|------|
| `error: externally-managed-environment` | PEP 668 保护 | 创建 `.venv`：`python3 -m venv .venv` |
| `Running xmake as root is extremely dangerous` | xmake 拒绝 root | 所有 xmake 命令加 `XMAKE_ROOT=y` 或 `--root` |
| `fatal error: infinicore/dtype.hpp` | InfiniCore 未安装 | 先编译安装 InfiniCore |
| `g++: fatal error: Killed` | 编译 OOM | xmake build 加 `-j4` 限制并行 |
| `Invalid option: -j` | xmake install 不支持 -j | 只有 build 命令加 -j |
| `cudaErrorInsufficientDriver` | CUDA Toolkit > 驱动版本 | 降级 CUDA Toolkit |
| `Driver/library version mismatch` | NVML 库版本不一致 | 清除高版本 NVML 包 |
| `FlashAttention is not enabled in this build` | InfiniCore 未编译 FlashAttn | 先用 default attn 跑 |
| `fatal error: cutlass/bfloat16.h` | cutlass 子模块未初始化 | `git clone --depth 1 ... third_party/cutlass` |
| GPU count = 0 | CUDA 版本/驱动不匹配 | 检查驱动 + CUDA 版本 + InfiniCore GPU 后端编译 |
| `test_infer.py` 输出乱码 | LLM.generate() 离线路径有 bug | 用 server API，避免离线 LLM 类 |
| OOM | num-blocks 太大 | 减小 num-blocks 或加 GPU |
| PyTorch CUDA 初始化失败 | torch cu130 不兼容驱动 12.8 | 装 torch cu124 |
